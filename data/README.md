# `data/hira_stats.csv` — 빅5 병원 수술 정량지표 (실데이터 적재용)

이 CSV는 **병원별 × 질환(KCD)별 수술건수·사망률·합병증률·재원일수**의 1순위 소스입니다.
이 값들은 공개 REST API로 사실상 제공되지 않으므로(병원평가정보서비스 API는 "등급"만 제공),
HIRA가 공개한 **통계/적정성평가 보고서**에서 수치를 추출해 여기에 손으로 적재합니다.

행이 채워지면 `backend/api/hira.py` 가 그대로 읽어 실데이터로 표시하고(`isEstimate=False`),
행이 없으면 추정치(`isEstimate=True`, "추정치" 라벨)로 폴백합니다 — 가짜를 실데이터로 위장하지 않습니다.

## 컬럼

| 컬럼 | 의미 | 예 |
|---|---|---|
| `hospital_id` | 병원코드: `snuh` `amc` `smc` `sev` `snubh` | `amc` |
| `kcd_code` | 질환 KCD 코드(앱이 질환→KCD 매핑) | `C16` (위암) |
| `annualSurgeries` | 연간 수술건수 | `1850` |
| `annualCases` | 연간 진료(환자)건수 | `4200` |
| `mortalityRate` | 수술 사망률(%) | `0.8` |
| `complicationRate` | 합병증률(%) | `3.2` |
| `readmissionRate` | 재입원률(%) | `4.1` |
| `avgLOS` | 평균 재원일수(일) | `9` |
| `trend` | 연도별 추이 `YYYY:건수\|YYYY:건수\|...` (가용연도만, 개수 자유) | `2021:1511\|2022:1586\|2023:1487` |
| `dataYear` | 대표 기준연도(미상이면 공란) | `2023` |
| `source` | 출처 문구(UI 노출). 근사치는 "약" 명시 | `서울아산병원 2025 Outcomes Book (위암 수술)` |

> 사망률/합병증률/재입원률/평균재원/연간진료는 **병원별×질환별로 비공개**라 `0`으로 둠 → UI에서 "—"로 표시.
> 수술건수(annualSurgeries)+trend만 검증 출처로 채운다. 근사치는 값을 넣되 `source`에 "약"을 명시.

> `kcd_code` 는 프론트의 질환→KCD 매핑과 일치해야 행이 조회됩니다.
> 매핑 확인: `backend/` 의 질환 매퍼(예: 위암=C16, 대장암=C18, 폐암=C34, 유방암=C50,
> 간암=C22, 전립선암=C61, 자궁암=C53/C54, 갑상선암=C73).

## 데이터 출처 (실수치 채울 곳)

출처 우선순위(이미 적용된 정책):
1. **HIRA 적정성평가 보고서** — 단, 조사 결과 **병원별 대상건수는 비공개(등급만 공개)** 라 수술건수 적재엔 부적합.
2. **각 병원 공식 Outcomes Book / 암병원 연보** — 현재 적재된 빅5 수치의 1차 근거.
   - 서울아산: https://cancer.amc.seoul.kr (Outcomes Book PDF, 막대그래프 실측치)
   - 분당서울대: https://www.snubh.org/outcomesbook.do (8th Edition)
   - 삼성서울/서울대: 암센터 공시값(일부 근사 "약")
3. **위 출처를 인용한 의료 언론 기사**.

> 세브란스(신촌)는 공개 연간 수술건수가 정성표현·누적치뿐이라 **검증 행 없음** → 자동 추정치(isEstimate) 폴백.
> 추가/갱신 시 반드시 **출처+기준연도를 `source`에 명시**(검증 가능성 = 이 프로젝트의 핵심 원칙).

## 실제 적재 예시 (현재 파일에 들어있는 행)

```
snubh,C34,753,0,0,0,0,0,2022:911|2023:1019|2024:753,2024,분당서울대병원 2025 Outcomes Book 8판 (근치적 폐절제술)
smc,C50,3000,0,0,0,0,0,2019:2735|2020:2794|2021:3000,2021,삼성서울병원 (whosaeng 2021·국내 최초 연 3000건)
```
